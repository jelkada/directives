Directives basics:

  • Directives are markers on a DOM element that tell Angular's HTML compiler to manipulate the DOM by adding, removing or modifying DOM elements.
  • Before a view is displayed, Angular evaluates or compiles the directives to modify the HTML elements and the DOM.
  • Like a component, a directive is essentially a function that executes when the Angular compiler finds it in the DOM.
  • In templates, directives typically appear within an element tag as attributes.
  • There are two kinds of directives in Angular.
  • (1) Structural directives — change the DOM layout by adding and removing DOM elements (NgFor, NgIf).
  • (2) Attribute directives — change the appearance or behaviour of an element (NgStyle, ngClass).
  • A Component is sometimes referred to as a "directive with a dedicated template".
  • The star * before the directive indicates to Angular that we are using a structural directive.
  • With the * shortcut Angular transforms the code to an <ng-template> element
  • For example, <div *ngIf="!userActive"> is equivalent to <ng-template [ngIf]="!userActive">

Usage:

  • A directive is a class with a @Directive() decorator which must have a selector property for the directive's name
    Ex:@Directive({ selector: ‘[textHighlight]' })
  • Define in the attribute directive class constructor a reference to the DOM element where the directive will be added.
    ex: constructor(private elementRef: ElementRef)...
  • For arrtibute directive use the ElementRef type in the constructor to access the element's attributes
    constructor(private elementRef: ElementRef) { }
  • For structural directive use the TemplateRef and ViewContainerRef types in the constructor to access the element's DOM node
    constructor(private templateRef: TemplateRef<any>, private viewContainerRef: ViewContainerRef) { }
  • The templateRef variable is the "what" html code/template to inject and viewContainerRef variable is the "where" to inject or add the HTML code

Components vs. Directives:

  • Components look like HTML tags while directives look like HTML elements.
  • Components must have a dedicated HTML template or @View, whereas directives cannot have one.
  • Components have their own view (hierarchy in DOM elements) with attached behaviour.
  • Directive supplement an existing DOM elements, whereas components create new DOM elements.
  • Directive classes use the @Directive() decorator, and components classes use the @Component() decorator.
  • Components use lower-case dash-delimited naming convention, whereas directives are usually named using camelCase notation.
  • Component is a directive which uses shadow DOM to create encapsulated visual behaviour.
  • Only one component can be present per DOM element, whereas many directives can be used in one DOM element.
  • @Component decorator is actually a @Directive decorator extended with template-oriented features.
